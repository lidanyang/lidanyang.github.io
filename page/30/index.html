¶LCP 5. 发 LeetCoin

Difficulty: 困难

力扣决定给一个刷题团队发LeetCoin作为奖励。同时，为了监控给大家发了多少LeetCoin，力扣有时候也会进行查询。

该刷题团队的管理模式可以用一棵树表示：

  1. 团队只有一个负责人，编号为1。除了该负责人外，每个人有且仅有一个领导（负责人没有领导）；
  2. 不存在循环管理的情况，如A管理B，B管理C，C管理A。

力扣想进行的操作有以下三种：

  1. 给团队的一个成员（也可以是负责人）发一定数量的LeetCoin；
  2. 给团队的一个成员（也可以是负责人），以及他/她管理的所有人（即他/她的下属、他/她下属的下属，……），发一定数量的LeetCoin；
  3. 查询某一个成员（也可以是负责人），以及他/她管理的所有人被发到的LeetCoin之和。

输入：

  1. N表示团队成员的个数（编号为1～N，负责人为1）；
  2. leadership是大小为(N - 1) * 2的二维数组，其中每个元素[a, b]代表b是a的下属；
  3. operations是一个长度为Q的二维数组，代表以时间排序的操作，格式如下：
    1. operations[i][0] = 1: 代表第一种操作，operations[i][1]代表成员的编号，operations[i][2]代表LeetCoin的数量；
    2. operations[i][0] = 2: 代表第二种操作，operations[i][1]代表成员的编号，operations[i][2]代表LeetCoin的数量；
    3. operations[i][0] = 3: 代表第三种操作，operations[i][1]代表成员的编号；

输出：

返回一个数组，数组里是每次查询的返回值（发LeetCoin的操作不需要任何返回值）。由于发的LeetCoin很多，请把每次查询的结果模1e9+7 (1000000007)。

示例 1：

限制：

  1. 1 <= N <= 50000
  2. 1 <= Q <= 50000
  3. operations[i][0] != 3 时，1 <= operations[i][2] <= 5000

lucas定理求卡特兰数

发表于 2019-11-17 | 更新于: 2019-11-23 | 分类于 算法 ， 数学

¶Lucas定理用来求大组合数

  定理表达式为:

  Lucas定理可以按照下面这种方式来理解Lucas定理:A、B是非负整数，p是质数。AB写成p进制：A=a[n]a[n-1]...a[0]，B=b[n]b[n-1]...b[0]。则组合数C(A,B)与C(a[n],b[n])*C(a[n-1],b[n-1])*...*C(a[0],b[0]) modp同余。

  求组合数C(a,b)%p时可以利用C(a,b)%p =a! / ( b! * (a-b)! ) mod p=( a! / (a-b)!) * ( b! )^(p-2) mod p来求，这个公式可以用费马小定理推出。

  最后可以用Lucas(n,m,p)=c(n%p,m%p)*Lucas(n/p,m/p,p)递归求解C(n,m)%p。

¶状态压缩dp

  调试该程序时踩的坑:1.障碍物的位图vector<int> board(n+1)的大小要设置比总行数多1，因为在if(!check(state, j)&&!check(board[i], j)&&!check(board[i+1], j))检查当前行能否竖放时，会涉及到下一行的board值，而当i为n-1时，board访问的下标就可能取到n。

¶二分图匹配

  先将每一块染色，然后对黑白配对，不断从黑块寻找增广路，找到一条增广路既可以多放一块骨牌，找完所有增广路，就得出答案。

¶轮廓线动态规划

解释:

  1. block & state != block这句中block代表障碍物窗口的位图(为1代表窗口中该位置有障碍物)，state表示当前棋盘窗口位图(为1表示窗口中该位置有障碍物或者骨牌块)，这句判断是将该有障碍物的位置却没有障碍物的state略去，因为合法的state障碍物位置处一定是1。
  2. 一个合法的state一共可能导出三种下一个state窗口，即竖着放，横着放或者不放。要考虑这三种情况。
  3. 为了节省空间采用了滚动dp，next_dp是当前轮次每个state对应的块数，dp是上一轮次每个state对应的块数，每次记得要更新block位图，并且要把next_dp赋给dp，以充当下一轮次中的上轮dp的结果。
