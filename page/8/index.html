解法

https://www.geeksforgeeks.org/tomohiko-sakamotos-algorithm-finding-day-week/

根据公式直接计算，非常简单，前提是1年1月1日是星期1。大致思路是：

  1. 365%7是1，所以不考虑闰年，每隔一整年，星期只加1，即year%7。
  2. 再考虑闰年366%7是2，所以比1.中多走了闰年个数个天，即year/4 - year/100+year/400。
  3. 最后一年月数小于3时和大于3时分开计算，大于2时2月按27天算，小于3时则year的闰年不考虑。假如year是闰年，year + year/4 - year/100 + year/400 + t[month-1] + day中year和year/4 - year/100 + year/400各自会比不考虑year的闰年加1，刚好比27多2补上了缺失的天数；假如year不是闰年，则year + year/4 - year/100 + year/400 + t[month-1] + day会比不考虑闰年总共多1，加上27是28符合非闰年2月的天数。

782. Transform to Chessboard

Difficulty: Hard

An N x N board contains only 0s and 1s. In each move, you can swap any 2 rows with each other, or any 2 columns with each other.

What is the minimum number of moves to transform the board into a “chessboard” - a board where no 0s and no 1s are 4-directionally adjacent? If the task is impossible, return -1.

Note:

  • board will have the same number of rows and columns, a number in the range [2, 30].
  • board[i][j] will be only 0s or 1s.

解法

  1. 先用board[0][0] ^ board[i][0] ^ board[0][j] ^ board[i][j]判断数据合法性，即是否任意两行互为取反或者互等，另一种说法是不同的行向量有两个，不同的列向量的个数也只有两个。其实只要不同的行向量有两个的话，那么不同的列向量一定也只有两个，因为翻转任意一行，不会改变两个列向量之间的关系。
  2. 计算其中一个行向量和列向量的个数，要满足不同的行向量的个数能大致相等；
  3. 计算不在正确位置上的0,1的个数，这个数一定要是个偶数；
  4. 根据n的奇偶性，分情况计算总共的不在其位的0,1的个数，将这个数除以2得出交换次数最小值，因为每一次有效的交换（,奇1和偶0或奇0和偶1）会使这个数减2，所以要除以2；

  • 1 <= arr[i] <= 20000
  • For each query, 0 <= left <= right < len(arr)
  • For each query, 2 * threshold > right - left + 1
  • The number of queries is at most 10000

题解

¶线段树

  在查询函数中需要注意一个易错点：int find(int root, int left, int right)返回的是left到right之间的主元素，所以find(root<<1, left, right);和find(root<<1|1, left, right);直接返回即可，而find(root<<1, left, ans[root<<1].r);和find(root<<1|1, ans[root<<1|1].l, right);的返回值要在[left，right]这个区间检查，而不是[ans[root].l, ans[root].r]。
