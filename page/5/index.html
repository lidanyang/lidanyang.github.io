87. Scramble String

发表于 2019-05-31 | 分类于 leetcode

¶题目描述

将一个字符串看成是二叉树，如下：

        great
          +
          |
    +-----+----+
    |          |
    +          +
   gr         eat
   +           +
+--+--+     +--+---+
+     +     +      |
g     r     e    +-+-+
                 +   +
                 a   t

  选择任一个非叶节点，交换它的两个子节点，重复上述过程，得到的树对应的字符串，为一个字符串的scramble string。写程序判断两个字符串是否互为sramble string。

¶思路

  对于一个字符串s1组成的树的根节点s1的子节点是s1[:i]和s1[i:]，并将对s1进行scramble的过程简记为scr(s1)。则scramble string可能是scr(s1[:i])+scr(s1[i:])或者scr(s1[i:])+scr(s1[:i])。即：

    scr(s1) == scr(s1[:i]) + scr(s1[i:]) or scr(s1) == scr(s1[i:]) + scr(s1[:i])

  问题可以转化为两个子问题：

  • s2[:i]和s2[i:]是不是s1[:i]和s1[i:]的scramble string；
  • s2[:-i]和s2[-i:]是不是s1[i:]和s1[:i]的scramble string；
      显然可以用递归求解。

84. Largest Rectangle in Histogram

发表于 2019-05-29 | 分类于 leetcode

¶题目描述：求直方图最大矩形面积

¶解法

  1. 暴力解法
    求每个区间的最小值和区间长度的乘积

$$ \min_{ 0 <= i < size, i < j < size}(\min_{ i <= t < j}(heights[t]) * (j - i)) $$

  1. 单调栈
    1. 循环入栈，栈中存放坐标；
    2. 遇到小元素出栈，并计算面积，直到遇到第一个比当前元素小的栈顶元素；
    3. 在heights最后加一个0，方便计算边界情况

¶代码实现

  1. 我的代码
  1. leetcode上100%代码
  1. discuss里发现的一种更容易理解的实现

思路是求出每个bar往左或者往右大于等于当前bar值的bar的个数（即包含当前bar的最大矩形的宽度），然后用当前bar值乘以该宽度就是包含当前bar的最大矩形面积。

代码中利用了height[j - 1]如果大于等于height[j]那么，大于等于height[j - 1]的bar都大于height[j]，跳过这些不用比较的元素，直接从j-left[j]位置重复上述过程，直到height[j] < height[i]。

¶总结

leetcode上的jddymx用户给出的栈做法的解释：
　　(以下的论述都用下标i代替height[i]来讨论)

  1. 令$S_{i}$是以height[i]为最低点的最大矩形面积；
  2. 栈易证有两点性质：
    • 从左往右的下标以height递增的顺序存在栈中；
    • 任意栈中的两个下标i和j之间的k都比j对应的高度要大；
  3. 注意到每个bar都至少如一次栈出一次栈，入栈的时候可能会“逼”之前的bar出栈，而出栈的时候会计算$S_{i}$
  4. 满足2中两点性质的栈中的i，当i的下一个bar（记为k）小于i的高度，则栈顶元素弹出直到遇到一个比k小的bar，同时计算$S_{i}$。假设弹出j（j比k大），那么$S_{j}$的左端点即为栈顶的bar，右端点刚好就是k对应的bar，此时可计算得出$S_{j}$。由于每个bar都会弹出一次，因此所有的情况都会计算到。故该方法可以求出答案。

142. Linked List Cycle II

发表于 2019-05-28 | 分类于 leetcode

求有环链表的环节点

  之前有一道题是判断链表是否有环，这一道题进一步求如果有环那么环从哪个节点开始。

¶关键点

  假设h和t两个指针，h每次走一步，t每次走两步，那么两个指针必定会在环上相遇；此时h走的步数比t刚好少整数倍环的长度，又h的路程是t的一半，所以h走的路程也是换的长度的整数倍。
  设进入环之前的小尾巴的长度为l，环的周长为c的话。建立以环节点为原点，顺时针方向的坐标系，则kc-l是相遇点的坐标。
本题关键在于能意识到从相遇点kc-l再走l长度就会回到环节点（相当于绕环k圈）。如果再相遇之后，让一个指针h从相遇点出发，让另一个指针t回到head，然后两者都每次走一步，由上面的分析，当走l步的时候，h会回到环节点，t会到达环节点并即将进入环，两者会在环节点相遇。

¶代码实现

¶总结

  我的做法：将环从相遇点断开，问题转化成求两个链表的交点。分别求两个链表的长度，求长度差的绝对值，分别设两个指针从两个链表头开始遍历，并且预先让长链表的指针先走一个长度差，然后两指针必然就会在交点处相遇。
  做的时候也想过是不是有什么技巧，但是思考的时候只考虑一个比另一个多走了整数倍环的长度，忘了两者总路程还有一个2倍的关系。说明思考还是欠全面，以后想问题要先把条件挖掘充分了再分析。

72. Edit Distance

发表于 2019-05-28 | 分类于 leetcode

求两个单词的最短"距离"

主要思想:

word1向word2转化时，如果从前往后调整word1以靠近word2的话，那么word1的最后一个字符的处理只有如下三种情况：

  1. word1的末尾添加一个和word2末尾相同的字符，则问题转化为求word1和word2[:-1]的最短距离；
  2. word1的末尾替换成和word2末尾相同的字符，则问题转化为求word1[:-1]和word2[:-1]的最短距离；
  3. word1的末尾删除一个字符，则问题转化为求word1[:-1]和word2的最短距离；

代码实现

  1. 宽度优先搜索（代码来自Accepted Solutions Runtime Distribution)
  1. 动态规划

总结

  1. 用宽度优先的速度比动态规划明显要快；
  2. 我最初的想法是先求最长公共子串，然后调整word1中公共子串之外的部分，然而这需要知道最长公共子串的具体位置，不好实现就pass了（懒）；
  3. 为什么找最短要按这个步骤？或者说为什么以循环对word1最后一个字符加删改的方式搜索下去，如果遇到word2那么搜索的深度就是最短距离？严格的数学证明并没有看到，可能只是凭感觉或者说是直觉，觉得照这个步骤来搜索就是最短距离，但是还是感觉心里不踏实，接受程度60%，待以后看到更清晰的解释；
  4. 貌似很多程序都是凭感觉写出来的，不需要深究为什么，记住就行了，程序员并不需要成为数学家，只需要成为玄学家！
