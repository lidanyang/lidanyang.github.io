204. Count Primes

发表于 2019-06-08 | 更新于: 2019-06-25 | 分类于 leetcode

题目描述

204. Count PrimesCopy for Markdown

Difficulty: Easy

Count the number of prime numbers less than a non-negative number, n.

Example:

基本思路

  求小于n的所有素数的个数，可以遍历并判断小于n的每个数是不是素数，但是这样效率很低，而且会有重复的判断。比如你已经判断10不是素数了，那么所有10的倍数的数都不用判断了。根据这种想法，可以采取一种叫筛法的求素数方法。分配一个大小为n的数组，数组值记录是否为素数，是为True，否则为False。2是第一个素数，将所有2的倍数（不包括2自身）标记为False，然后从2开始找下一个素数a（对应数组值为True），重复上述标记倍数的过程，直到找不到下一个素数为止。预处理之后，从头遍历数组，记录True的数目，即为素数的个数。
  看似筛法已经很快了，但是还有优化的余地。考虑一下标记素数p的倍数(p*2, p*3,…)，仔细看一下可以发现p*q(q < p)的数，在标记q的倍数的时候已经被标记过了（因为筛法找素数是从小往大找的，所有小于p的素数在p之前已经经历过标记过程了），因此所有小于p*p的数都可以不用标记，直接标记p*p,p*(p+1),…。

题外话

  关于判断一个数是不是素数其实也有一些技巧，这里总结一下，整理一下知识树。

  对一个数N，要判断N是不是素数，暴力解法是判断N能不能被从2到N-1的数整除，如果N不能被所有[$\sqrt N$ ] + 1整除，那么N肯定也不能被大于[$\sqrt N$ ] + 1的数整除。因为N=M*L的话，M和L一定有一个大于等于[$\sqrt N$ ] + 1，另一个小于等于[$\sqrt N$ ] + 1，一个小于[$\sqrt N$ ] + 1的因子一定对应一个大于[$\sqrt N$ ] + 1的因子。所以只需要判断所有小于[$\sqrt N$ ] + 1的数能不能整除N。

Given a matrix consisting of 0s and 1s, we may choose any number of columns in the matrix and flip every cell in that column.  Flipping a cell changes the value of that cell from 0 to 1 or from 1 to 0.

Return the maximum number of rows that have all values equal after some number of flips.

Note:

  1. 1 <= matrix.length <= 300
  2. 1 <= matrix[i].length <= 300
  3. All matrix[i].length's are equal
  4. matrix[i][j] is 0 or 1

解法

逆向思考：对于结果全0和全1的行，不管列怎么翻转，所有全0的行原来相同，列翻转后也保持相同，所有全1的行原来相同，列翻转后也保持相同，所有全0和全1的行也都保持相反。如果将每一位都相同或相反叫做本质相同的行，所以全0或全1最多的数目即为本质相同的行的最多数目。

总结

对于这种题，要擅长观察变化中的不变量。首先找对象中的一些关系（相等，不等之类的），然后用题中的处理方法对对象转化，观察这些对象中的关系的变化和不变，以及这些变化和不变量和所求结果的关联。

                 +   +
                 a   t

  选择任一个非叶节点，交换它的两个子节点，重复上述过程，得到的树对应的字符串，为一个字符串的scramble string。写程序判断两个字符串是否互为sramble string。

¶思路

  对于一个字符串s1组成的树的根节点s1的子节点是s1[:i]和s1[i:]，并将对s1进行scramble的过程简记为scr(s1)。则scramble string可能是scr(s1[:i])+scr(s1[i:])或者scr(s1[i:])+scr(s1[:i])。即：

    scr(s1) == scr(s1[:i]) + scr(s1[i:]) or scr(s1) == scr(s1[i:]) + scr(s1[:i])

  问题可以转化为两个子问题：

  • s2[:i]和s2[i:]是不是s1[:i]和s1[i:]的scramble string；
  • s2[:-i]和s2[-i:]是不是s1[i:]和s1[:i]的scramble string；
      显然可以用递归求解。

84. Largest Rectangle in Histogram

发表于 2019-05-29 | 更新于: 2019-06-25 | 分类于 leetcode

¶题目描述：求直方图最大矩形面积

¶解法

  1. 暴力解法
    求每个区间的最小值和区间长度的乘积

$$ \min_{ 0 <= i < size, i < j < size}(\min_{ i <= t < j}(heights[t]) * (j - i)) $$

  1. 单调栈
    1. 循环入栈，栈中存放坐标；
    2. 遇到小元素出栈，并计算面积，直到遇到第一个比当前元素小的栈顶元素；
    3. 在heights最后加一个0，方便计算边界情况

¶代码实现

  1. 我的代码
  1. leetcode上100%代码
  1. discuss里发现的一种更容易理解的实现

思路是求出每个bar往左或者往右大于等于当前bar值的bar的个数（即包含当前bar的最大矩形的宽度），然后用当前bar值乘以该宽度就是包含当前bar的最大矩形面积。

代码中利用了height[j - 1]如果大于等于height[j]那么，大于等于height[j - 1]的bar都大于height[j]，跳过这些不用比较的元素，直接从j-left[j]位置重复上述过程，直到height[j] < height[i]。

¶总结

leetcode上的jddymx用户给出的栈做法的解释：
　　(以下的论述都用下标i代替height[i]来讨论)

  1. 令$S_{i}$是以height[i]为最低点的最大矩形面积；
  2. 栈易证有两点性质：
    • 从左往右的下标以height递增的顺序存在栈中；
    • 任意栈中的两个下标i和j之间的k都比j对应的高度要大；
  3. 注意到每个bar都至少如一次栈出一次栈，入栈的时候可能会“逼”之前的bar出栈，而出栈的时候会计算$S_{i}$
  4. 满足2中两点性质的栈中的i，当i的下一个bar（记为k）小于i的高度，则栈顶元素弹出直到遇到一个比k小的bar，同时计算$S_{i}$。假设弹出j（j比k大），那么$S_{j}$的左端点即为栈顶的bar，右端点刚好就是k对应的bar，此时可计算得出$S_{j}$。由于每个bar都会弹出一次，因此所有的情况都会计算到。故该方法可以求出答案。
