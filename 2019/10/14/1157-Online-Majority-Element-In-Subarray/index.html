1157. Online Majority Element In Subarray

Difficulty: Hard

Implementing the class MajorityChecker, which has the following API:

  • MajorityChecker(int[] arr) constructs an instance of MajorityChecker with the given array arr;
  • int query(int left, int right, int threshold) has arguments such that:
    • 0 <= left <= right < arr.length representing a subarray of arr;
    • 2 * threshold > right - left + 1, ie. the threshold is always a strict majority of the length of the subarray

Each query(...) returns the element in arr[left], arr[left+1], ..., arr[right] that occurs at least threshold times, or -1 if no such element exists.

Example:

Constraints:

  • 1 <= arr.length <= 20000
  • 1 <= arr[i] <= 20000
  • For each query, 0 <= left <= right < len(arr)
  • For each query, 2 * threshold > right - left + 1
  • The number of queries is at most 10000

题解

¶线段树

  在查询函数中需要注意一个易错点：int find(int root, int left, int right)返回的是left到right之间的主元素，所以find(root<<1, left, right);和find(root<<1|1, left, right);直接返回即可，而find(root<<1, left, ans[root<<1].r);和find(root<<1|1, ans[root<<1|1].l, right);的返回值要在[left，right]这个区间检查，而不是[ans[root].l, ans[root].r]。
