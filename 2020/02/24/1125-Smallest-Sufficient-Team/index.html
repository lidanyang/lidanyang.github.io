¶1125. Smallest Sufficient Team

Difficulty: Hard

In a project, you have a list of required skills req_skills, and a list of people.  The i-th person people[i] contains a list of skills that person has.

Consider a sufficient team: a set of people such that for every required skill in req_skills, there is at least one person in the team who has that skill.  We can represent these teams by the index of each person: for example, team = [0, 1, 3] represents the people with skills people[0], people[1], and people[3].

Return any sufficient team of the smallest possible size, represented by the index of each person.

You may return the answer in any order.  It is guaranteed an answer exists.

Constraints:

  • 1 <= req_skills.length <= 16
  • 1 <= people.length <= 60
  • 1 <= people[i].length, req_skills[i].length, people[i][j].length <= 16
  • Elements of req_skills and people[i] are (respectively) distinct.
  • req_skills[i][j], people[i][j][k] are lowercase English letters.
  • Every skill in people[i] is a skill in req_skills.
  • It is guaranteed a sufficient team exists.

¶Solution

¶法一 dp

  状态压缩dp，dp的过程中用prv数组维护状态转移的信息用来还原最小队伍。每个技能对应状态的一个位，每个人的技能也可以用一个数（每一位对应是否掌握某项技能）来表示。dp[i]中代表当前队伍的技能的掌握情况为i时队伍的最少人数，prv[i]当前队伍的技能的掌握情况为i时队伍的最少人数为dp[i]的上一个入选的人和上一个状态。

下面的dp解法和我自己的解法类似，但感觉维护状态转换信息的方式比我的更好。

同样是dp，用uint64_t的每一位来代表一个人，最后统计(1<<n) - 1对应的值哪些位上有1即可。

¶法二 bfs

最快的解法是bfs，将寻找最小队伍看成寻找0（没有技能）到(1<<n)-1（所有技能都有）的最短路的问题。队列用有0号技能的人初始化，然后每轮bfs都至少多添加一项新技能，直到所有技能都添加。另外，这个解法的位操作运用的炉火纯青，值得借鉴。
